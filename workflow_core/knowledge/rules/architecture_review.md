# Protocol: Architecture & API Specifications

1. **Granularity**: Perform one sub-task at a time (e.g., 2.2.1.1).
2. **Review First**: Do NOT merge immediately.
3. **Presentation**: Present a detailed report used `notify_user` with:
    - What was done.
    - Location of documents.
    - Design choices & Alternatives (Pros/Cons).
4. **Gate**: WAIT for explicitly "Green Light" from user before marking as Done.
